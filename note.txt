Тех долг:
1. Модальное окно для создания ППРа - наименование, год, цех и т.д. и т.п. Подумать про роли
2. В api (в частности при создании ППР) используется избыточное кол-во параметров. Не надо столько отправлять данных. Большинство из данных должно делаться на сервере
3. pprtablestatusupdate - можно переписать как серверный компонент. Ибо все взаимодействия кнопочные, отображает данные чисто с сервера и т.д. и т.п. Хотя если для кнопок сделаю состояния (анимации загрузки и т.д.), то тогда надо будет кнопку выносить на клиентскую часть

Идеи:
1. Когда создаешь ППР, или копируешь чужой, то автоматически все значения цеха меняются на то, какой цех себе его скопировал
